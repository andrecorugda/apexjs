Overview #

apex ships with @apex-stack/core. Install it globally (npm i -g @apex-stack/core) and apex works in any project; otherwise it's a project-local binary wired to npm scripts (npm run dev) or run via npx apex.

CommandWhat it does
apex newScaffold a new app, install dependencies, and init git.
apex devStart the development server (SSR + hydrate, API + MCP, hot reload).
apex buildPrerender pages to deployable HTML + client bundles.
apex startRun a production build made with apex build --server.
apex makeGenerate a page, component, API route, store, layout, service, test, middleware, model, migration, or auth.
apex addCopy a themeable UI-kit component into your project.
apex themeWrite/update your theme tokens (colors, radius, fonts) in your CSS.
apex upgradeUpdate the CLI, project deps, scaffold files, and the .alpine extension — one command.
apex migrateApply pending SQL migrations (or roll back).
apex mcpInspect the local MCP server — list or call tools.
apex testRun your tests with Vitest.

apex new #

Scaffold a new Apex JS app in a new directory — then install dependencies and initialize a git repo.

apex new [dir] [--no-install] [--no-git] [--no-vscode]
Argument / flagDefaultDescription
dir (positional)apex-appTarget directory for the new app.
--no-installinstallsSkip installing dependencies.
--no-gitinits gitSkip initializing a git repository.
--no-vscodepromptsSkip the offer to install the .alpine VS Code extension.

No global install? npm create apexjs@latest my-app runs the same scaffolder via npx, then use npm run dev.

apex dev #

Start the Apex JS development server.

apex dev [root] [--port <n>] [--islands]
Argument / flagDefaultDescription
root (positional).Project root.
--port3000Port to listen on.
--islandsfalseRender in islands mode (static-first). See Islands.
 
apex dev
apex dev --port 4000
apex dev --islands

apex build #

Prerender pages to deployable HTML plus per-page client bundles.

apex build [root] [--outDir <dir>] [--islands] [--server]
Argument / flagDefaultDescription
root (positional).Project root.
--outDirdistOutput directory.
--islandsfalseStatic-first islands mode (zero-JS static site).
--serverfalseBuild a Node server (dynamic routes + API/MCP). Pair with apex start.
 
apex build            # prerender + hydrate → static dist/
apex build --islands  # zero-JS static site (SSG)
apex build --server   # deployable Node server target

The three modes are explained in full on Build & deploy.

apex start #

Run a production build produced by apex build --server (no Vite).

apex start [dir] [--port <n>]
Argument / flagDefaultDescription
dir (positional)distBuild directory (must contain apex-manifest.json).
--port3000Port to listen on.
 
apex build --server && apex start

apex upgrade #

Bring an existing app up to date — non-destructively. One command handles the whole chain: it updates the global CLI if a newer one is published (then re-runs on it), bumps your @apex-stack/* dependencies and installs them, adds any new scaffold files (never overwriting yours), and updates the .alpine editor extension. 

apex upgrade [root] [--force] [--no-install] [--no-self]
Argument / flagDefaultDescription
root (positional).Project root.
--forcefalseRe-sync scaffold files that differ from the template (package.json is always preserved).
--no-install—Skip running the package manager after bumping dependencies.
--no-self—Don't update the global CLI first (skip the self-update check).
 
apex upgrade

Your files are never overwritten; package.json entries are only version-bumped. It's the one command to run after a new release.

apex make #

Generate a page, component, API route, store, layout, service, test, middleware, model, migration, or auth from a template.

apex make <kind> [name] [fields…] [--root <dir>] [--no-test]
Argument / flagDefaultDescription
kind (positional)—page | component | api | store | layout | service | test | middleware | model | migration | auth (required).
name (positional)—Name, e.g. about, Counter, todos. Required for every kind except auth.
fields (positional, model only)—Field specs like title:string, views:int, email:string! (trailing ! = NOT NULL).
--no-testgenerates a testSkip the co-generated test for service / api / model.
--root.Project root.
KindCreates
pagepages/<name>.alpine — a loader + x-data template + scoped style.
componentcomponents/<Name>.alpine — a small interactive component (PascalCase).
apiserver/api/<name>.ts — a defineApexRoute that's REST + MCP (+ a test).
storestores/<name>.ts — a defineStore of shared reactive state.
layoutlayouts/<name>.alpine — a wrapper with a <slot>.
serviceservices/<Name>Service.ts — a plain, testable class (+ a test).
testtests/<name>.test.ts — a Vitest spec.
middlewaremiddleware/<name>.ts — a defineMiddleware that runs on every request.
modelmodels/<name>.ts + server/api/<name>.ts + a migration (+ a test) — a defineModel (from @apex-stack/data) with a REST + MCP resource.
migrationdb/migrations/<stamp>_<name>.sql — an empty, reversible migration.
authserver/auth.ts + server/api/login.ts + server/api/logout.ts — sealed-cookie session auth (no name argument).
 
apex make page about
apex make component Counter
apex make api todos
apex make model post title:string body:text published:boolean!
apex make auth

Fails if the target file already exists (it won't overwrite your work).

apex add #

Copy a themeable UI-kit component's .alpine source into your project. Run it with no name to list the registry.

apex add [name…] [--all] [--force] [--root <dir>]
Argument / flagDefaultDescription
name (positional)—Component(s) to add, space-separated (e.g. button card modal). Omit to list what's available.
--allfalseAdd every component in the registry.
--forcefalseOverwrite existing component files.
--root.Project root.
 
apex add                 # list available components
apex add button card
apex add --all

apex theme #

Write or update your theme tokens (colors, radius, fonts) in a managed block of your app.css. With no flags it (re)writes the default tokens.

apex theme [--primary <c>] [--primary-dark <c>] [--secondary <c>] [--radius <r>] [--font-body <f>] [--font-title <f>] [--out <file>] [--root <dir>]
FlagDefaultDescription
--primary—Primary color (light), e.g. #4f46e5.
--primary-dark—Primary color (dark).
--secondary—Secondary color (light).
--radius—Border radius, e.g. 0.5rem.
--font-body—Body font stack.
--font-title—Title font stack.
--outyour app.cssTarget stylesheet.
--root.Project root.
 
apex theme
apex theme --primary '#4f46e5' --radius 0.5rem

apex migrate #

Apply pending SQL migrations from db/migrations/*.sql (idempotent) — or roll the last one(s) back. Resolves @apex-stack/data from your app.

apex migrate [--rollback] [--steps <n>] [--db <file>] [--driver <d>] [--url <url>] [--dir <dir>] [--root <dir>]
FlagDefaultDescription
--rollbackfalseRevert the most recent migration(s) instead of applying pending ones.
--steps1How many migrations to roll back (with --rollback).
--dbdata.dbSQLite file path (libSQL).
--driversqlitesqlite | postgres | pglite.
--url—Connection URL (Postgres) — overrides --db.
--dirdb/migrationsMigrations directory.
--root.Project root.
 
apex migrate
apex migrate --rollback
apex migrate --driver postgres --url $DATABASE_URL

See Data → Migrations.

apex mcp #

Inspect the local MCP server — list its tools or call one. Reuses the bundled MCP SDK (nothing to install).

apex mcp [--url <url>] [--call <name>] [--args <json>]
FlagDefaultDescription
--urlhttp://localhost:3000/mcpMCP endpoint URL.
--call—Name of a tool to call. Omit to list all tools.
--args{}JSON arguments for --call.
 
apex mcp                                       # list tools + signatures
apex mcp --call posts --args '{"slug":"hello-apex"}'

Requires a running dev server. More on AI-native → inspector.

apex test #

Run your test suite with Vitest (the runner Apex apps ship with). It's a thin wrapper — any extra arguments pass straight through to Vitest.

apex test [...vitest args]
apex test              # Vitest (watch mode, its default)
apex test run          # run once and exit
apex test --coverage   # with coverage

See Test Kit for the in-process API + MCP harness.

create-apexjs #

The scaffolder (a separate, unscoped package) — the entry point for a new project.

npm create apexjs@latest [dir]   # dir defaults to "apex-app"

It copies the default template, renames _gitignore → .gitignore, substitutes the project name, and prints next steps. See Getting started.
